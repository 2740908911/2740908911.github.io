2.[网鼎杯 2020 青龙组]AreUSerialz

据说是当年的签到题，作为一个新手，当然要玩一下

进去直接给了原码，先复制到本地，找反序列化点：

然后审计代码，同第一题，但没有wakeup()，如果反序列化直接会destruct()

但再反序列化前，需要先绕过 is_valid()中的 ascll检查（私有类需要%00）

在destruct()中，当 op === “2”时，会被强制为”1”，当op == “1” 时，沿着代码走下去发现只能写，不能读，无法找到flag；而当 op == “2”时，通过file_get_contents()函数可以读到filename，如图：

所以我们的目标1.绕过 op === “2”时的强制转换 2.绕过私有属性时的ascll检测

解决方法：

1.当 op === “2”时才会转换，所以当op == “2”时，不会强制转换，因为后者没有对类型强制要求，所以只需让 op = 2 （int型），在判断时不会因为类型而引起强制转换，即绕过。

4.[安洵杯 2019]easy_serialize_php

进入网站就能看到原码，先复制到本地

先审计代码，在31行找到序列化函数，在38行找到但反列化函数，且参数可控通过get传参

所以，根据提示先让$function = phpinfo，通过bp传参（或是直接在浏览器输入）

我先用的bp，出现代码后复制到本地，查找 flag、ag、fl、f1ag等等相关名词，成功找到文件

后来又试了一下网址直接传，大概浏览了一遍也能找到。

知道了文件名，无法直接读，所以准备用代码中的file_get_contents()函数读取

所以需要让 base64_decode($userinfo[‘img’]) = d0g3_f1ag.php ，根据百度进行base64加密，

就需要让$userinfo[‘img’] = ZDBnM19mMWFnLnBocA==

又因为$userinfo[‘img’]是由$serialize_info反序列化得到的

$serialize_info又是由$_SESSION序列化然后过滤得到的

但是如果对$_SESSION[‘img’]赋值，则会被sha1进行再次加密，且不可逆，无法得到答案

所以只能从过滤函数过滤序列化字符串处开始下手

此处参考了https://blog.csdn.net/qq_43622442/article/details/106003691

当中对对象逃逸问题有详细的说明：（我还在参悟这东西，实在头大）

然后输入后查看源码有注释：

把img文件名加密后一换，长度没变，就能得到flag
