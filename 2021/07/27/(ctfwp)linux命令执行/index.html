这类的题不算多，在这里稍微总结一下

# [GXYCTF2019]Ping Ping Ping

第一次做这种题，有思路但缺少经验，所以没做出来，下面是 wp。

进去界面什么都没有，只有 /?ip=

查看源码无果，抓包无果，尝试输入 /?ip=1

有结果 根据提示：ping

考虑是 linux 命令，尝试查看目录 /?ip=1;ls

<!--p1-->

根据结果 cat flag.php，结果不让看，提示猜测应该是过滤了空格

<!--p2-->

经过百度后，发现有以下几种绕过过滤空格的方式：

  1. ${IFS} 替换
  2. $IFS$1 替换
  3. ${IFS 替换
  4. %20 替换
  5. <和 <> 重定向符替换
  6. %09 替换

尝试第一种不行，(这里我提前做了下 FUZZ 测试，就不放图了，因为我看大佬都没用) 发现 {} 等等都被过滤了

于是用第二种

<!--p3-->

发现 flag 被过滤了，查阅资料后，发现两张方式最合适。

第一种最常规： 命令执行拼接

原理就是 linux 命令的拼接，让 a=g，然后将 flag 换为 fla$a

第二种最牛逼： 内联绕过

内联，就是将反引号内命令的输出作为输入执行。

ls 输出的内容直接被执行了

flag 如图：

<!--p4-->

另外我还学到了一些其他的方法，可能本题不适用，但也很常见。

单引号、双引号绕过，本题被 ban 了

反斜线绕过，本题被 ban 了

$1、$2 等和 $@绕过

如果 cat 被过滤，可以用下边的方法进行绕过～～

(1)more:一页一页的显示档案内容

(2)less:与 more 类似，但是比 more 更好的是，他可以[pg dn][pg up]翻页

(3)head:查看头几行

(4)tac:从最后一行开始显示，可以看出 tac 是 cat 的反向显示

(5)tail:查看尾几行

(6)nl：显示的时候，顺便输出行号

(7)od:以二进制的方式读取档案内容

(8)vi:一种编辑器，这个也可以查看

(9)vim:一种编辑器，这个也可以查看

(10)sort:可以查看

(11)uniq:可以查看

(12)file -f:报错出具体内容

本题还可以用

sh，bash 下编码

payload：

Y2F0IGZsYWcucGhw 是 cat flag.php 的 base64 编码

echo"str" | base64 -d
将base64编码的字符串str+换行 解码输出

|sh 是以 sh 运行，不是 bash。

具体参考下面这篇文章

Linux base64 -- 用 base64 编解码

另外，常见的管道符还有 || ; 如果做不出来可以尝试下其他的。
